# 更新日志 (CHANGELOG)

## [v3.0.0] - 2026-05-31
### ✨ 新功能
*   **📊 新增全网热榜聚合 `/u 热榜 <平台>`**: 支持哔哩哔哩、A站、HelloGitHub、网易云音乐、QQ音乐、微信读书共 6 个平台实时热榜，每 5 分钟自动更新。不带参数发送 `/u 热榜` 显示平台列表，输入错误平台名时给出提示。
*   **🖼️ 平台专属渲染卡片**: 新增独立热榜渲染引擎 `hotboard/renderer.py`，哔哩哔哩（Top 10 含封面）、网易云音乐、QQ音乐、微信读书（Top 10 含专辑/书籍封面）支持图片展示；HelloGitHub（Top 10）展示编程语言标签与项目简介；A站（Top 10）纯文字列表。各平台图标从本地 `hotboard/icons/` 读取 base64 嵌入，完全离线。
*   **📁 模块化热榜架构**: 新增 `hotboard/` 目录，`fetcher.py` 负责请求与错误处理，各平台解析逻辑独立维护于单独文件，后续扩展新平台只需新增一个解析文件与图标。
*   **⬇️ 文本降级支持**: 渲染服务故障时自动切换文本模式，各平台副信息（UP主、歌手、编程语言、作者等）及原始链接完整保留，不依赖 HTML 解析。
*   **🤖 新增 LLM 工具 `uapi_hotboard`**: 将热榜封装为 function calling 工具，LLM 可主动查询指定平台热榜并基于内容回答问题（如"今天B站热榜有没有关于AI的视频"），返回含标题、热度、副信息及原始链接的结构化文本，共 **16 个工具**。

### 🔧 其他改进
*   **📐 渲染截图留白修复**: `card_renderer.py` 改用 `display: table` + viewport 宽度锁定，渲染截图不再出现两侧及底部多余留白。
*   **🔁 渲染策略对齐**: `cmd_hotboard` 补全第二条渲染策略（`quality: 100 + ultra`），与 `_send_analysis_report` 保持四级策略一致。

## [v2.2.0] - 2026-05-03
### 🤖 LLM 工具调用支持
*   **🛠️ 新增 `llm_tools.py`**: 将 15 项查询指令封装为 AstrBot function calling 工具，LLM 可根据对话内容自动决策调用，用户无需手动输入指令前缀。
*   **📦 工具覆盖范围**: 天气、IP 查询、MC 服务器、MC 玩家、WHOIS、ICP 备案、快递追踪、万年历、GitHub、Steam、B站、答案之书、一言、Epic 免费游戏、随机字符串，共 **15 个工具**。
*   **🚫 排除不适合工具化的指令**: 随机图片、必应壁纸、每日新闻（返回图片，工具模式无意义）及二维码生成不纳入工具列表。


## [v2.1.2] - 2026-04-19
### 🔧 兼容性修复
*   **🏷️ 兼容 AstrBot 指令别名**: 新增 `_extract_arg` 辅助方法，修复通过 AstrBot WebUI「重命名指令」功能设置别名后，参数解析失败的问题。根本原因是 AstrBot 在别名触发时不改写 `event.message_str`，导致原有正则匹配不到命令前缀，参数被整段错误透传。现改为正则优先、剥离别名 token 兜底的双重策略，全部 13 处参数提取统一走此方法。

## [v2.1.1] - 2026-04-18
### 🛡️ 安全修复 & 问题修正

*   **🔒 补全鉴权头部** (`bing_daily.py`): 本地 session 创建时补充缺失的 `Authorization: Bearer` 头部，与其他所有模块保持一致。
*   **❌ 修复日期参数静默忽略** (`bing_daily.py`): 用户输入格式不合法的日期（如 `/u 必应 2023`）时，原版本会静默忽略并返回今日壁纸，现改为返回明确的格式错误提示，要求使用 `YYYY-MM-DD` 格式。
*   **📏 新增输入长度校验** (`mc_user.py`): 限制 MC 用户名最长 20 字符，防止超长字符串透传至上游 API。

## [v2.1.0] - 2026-04-12
### 🔧 渲染引擎修复
*   **🖼️ 修复渲染调用方式**: 修正 `html_render` 参数传递错误，改为正确的位置参数调用方式，与 AstrBot 渲染引擎兼容。
*   **🎯 多策略渲染**: 引入四级渲染策略（Ultra PNG → Ultra JPEG → High JPEG → 普通 JPEG），自动降级重试，提升出图成功率。
*   **✅ 图片合法性校验**: 渲染结果写入前校验 magic bytes（JPEG/PNG），防止服务器返回错误页被当作图片发送。
*   **📝 文本降级保护**: 渲染失败时自动 fallback 文本模式，修复因无 `yield` 导致消息被 LLM 接管的问题。
*   **📐 居中布局修复**: 修正卡片偏左问题，`body` 改用 `min-width: 100vw` 铺满视口后居中。
